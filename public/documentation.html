MyFlix API Documentation

Introduction

MyFlix is an API that provides information about movies and allows users to register, update their username, add and remove movies from their favorites list, and deregister. The API provides the following endpoints:

Endpoints

GET /movies

Returns a list of all movies in the database.

  • Endpoint URL: /movies
  • Method: GET
  • Response Format: JSON
  • Response Body Example: 
    [  {    "Title": "SuperBad",    "DirectorInfo": {      "Name": "Greg Mottola",      "Bio": "asfasfhasjf",      "BirthYear": 1972,      "DeathYear": 2000    },    "Genre": "Comedy",    "Description": "Two high school seniors embark on a wild night of adventure in an attempt to buy alcohol for a party."  },  ...]

GET /movies/:title

Returns information about a single movie by its title.

  • Endpoint URL: /movies/:title
  • Method: GET
  • Query Parameters:
    • title: the title of the movie to retrieve (string)
  • Response Format: JSON
  • Response Body Example: 
    {
  "Title": "SuperBad",
  "DirectorInfo": {
    "Name": "Greg Mottola",
    "Bio": "asfasfhasjf",
    "BirthYear": 1972,
    "DeathYear": 2000
  },
  "Genre": "Comedy",
  "Description": "Two high school seniors embark on a wild night of adventure in an attempt to buy alcohol for a party."
}

GET /movies/genre/:genreName

Returns a list of all movies in a specific genre.

  • Endpoint URL: /movies/genre/:genreName
  • Method: GET
  • Query Parameters:
    • genreName: the name of the genre to retrieve (string)
  • Response Format: JSON
  • Response Body Example: 
    [   {
          "Title": "SuperBad",
          "DirectorInfo": {
              "Name": "Greg Mottola",
              "Bio": "asfasfhasjf",
              "BirthYear": 1972,
              "DeathYear": 2000
          },
          "Genre": "Comedy",
          "Description": "Two high school seniors embark on a wild night of adventure in an attempt to buy alcohol for a party."
      }
  ]

GET /movies/directors/:directorName

Returns data about a movie based of director name

  • Endpoint URL: /movies/directors/:directorName
  • Method: GET
  • Response Format: JSON
  • Response Body Example: 
              {
            "Name": "Christopher Nolan",
            "Bio": "asfasfhasjf",
            "BirthYear": 1972,
            "DeathYear": 2000
        }

POST /Users

allows a new user to regiester

  • Endpoint URL: /users
  • Method: POST
  • Response Format: JSON
  • Response Body Example: 
              {
            "name": "John",
            "movie": [],
            "id": "4d00cf55-ee4d-400a-9b43-3c30131ffac6"
        }

POST movies/:id/:movieTitle

allow users to add a list of movies to there favourites

  • Endpoint URL: movies/:id/:movieTitle
  • Method: POST
  • Response Format: Plain Text
  • Response Body Example: 
              "movie has been added to user!"

PUT /users/:id

allow users to add a list of movies to there favourites

  • Endpoint URL: /users/:id
  • Method: PUT
  • Response Format: JSON
  • Response Body Example: 
              {
            "id": 1,
            "name": "Katie",
            "movie": []
        }

DELETE //movies/:id/:movieTitle

Allow users to remove a movie from there list

  • Endpoint URL: /movies/:id/:movieTitle
  • Method: DELETE
  • Response Format: Plain Text
  • Response Body Example: 
              Movie has been removed from list

DELETE /users/:id

Allow users to deregister

  • Endpoint URL: /users/:id
  • Method: DELETE
  • Response Format: Plain Text
  • Response Body Example: 
              User has been deleted
